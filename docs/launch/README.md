# Launch posts

Pre-written launch copy for v1.0. Keep updated for future major versions.

- [HN.md](HN.md) — Hacker News "Show HN" post
- [REDDIT.md](REDDIT.md) — r/LocalLLaMA + r/selfhosted variants

## Posting order (recommended)

1. **HN first** — Show HN gets the biggest single-shot audience but ranks
   on early upvote velocity. Post Tuesday or Wednesday, 9–10 a.m. PST.
2. **r/LocalLLaMA next** — local-model-savvy audience; tolerates technical
   detail. Post within 24 h of HN so cross-traffic compounds.
3. **r/selfhosted last** — broader audience; lead with the Docker compose
   one-liner.

## Pre-flight checklist

Before any post:

- [ ] Demo GIF is in README (not a 404 image)
- [ ] `npx @nandansai08/personal-ai` works from a clean directory on Node 20+
- [ ] GitHub Discussions enabled (already done)
- [ ] Issue templates surface a "Discussions" link (already done)
- [ ] Repo has topics + description set (already done)
- [ ] At least 10 `good first issue`-tagged issues open
- [ ] Pinned tweet / Mastodon post with the install command ready
- [ ] Personal availability to reply to comments for the first 4 h after posting

## Post-launch

- Tag every commenter on HN within an hour if they ask a question
- On Reddit, the first 10 comments set the thread tone — engage
- Open a Discussion category `Launch feedback v1.0` and pin
- Reply to issue and PR notifications within 24 h
